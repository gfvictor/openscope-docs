# 職歴（経験）

候補者によってこれまでの経歴は全く異なります。フォームは、各個人のキャリアの長さに適応する必要があります。

### これは何ですか？

モジュール式のフォームインターフェース（Field Arrays）で、候補者が「職歴」（フルタイム、パートタイムなど）の個別のブロックをワンクリックで無制限に追加または削除できるようにします。

### 何のためにあるのですか？

古いシステムでは、候補者が記入できる「過去の仕事のスペース」が3つなどに制限されていました。動的履歴は自由を提供するためのものです。初めて仕事に応募する候補者はブロックを完全に削除でき、ベテランは10個の異なるブロックを追加してキャリア全体を詳細に記述することができます。

### どのように使われますか？

フォームの最後に「職歴」セクションがあります：

1. 候補者は、事前定義されたブロックに経験を記入するか、職歴がない場合はフィールドを削除します。
2. 複数の経験がある場合は、**「職歴を追加」**ボタンをクリックして新しいブロックに記入します。
3. すでに記入したブロックを削除したい場合は、その特定のブロックの横にあるゴミ箱アイコンをクリックするだけです。

### 期待される結果

すっきりとした威圧感のないインターフェース。フォームは最初は短く、候補者が追加すべき情報を持っている場合にのみオンデマンドで拡大するため、コンバージョン率（フォームの入力と送信）が大幅に向上します。
