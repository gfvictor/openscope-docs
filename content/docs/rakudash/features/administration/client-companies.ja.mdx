# 顧客企業

### どのような機能ですか？

労働力の提供と配属のために派遣会社と契約しているすべての工場、倉庫、および企業の登録を含む完全な運営ハブです。

### 何のためのものですか？

顧客の住所や連絡先から、各部門の複雑なビジネスルールや給与計算に至るまでを管理し、タイムシートの締め切りプロセスや月次の請求業務を正確に整理するためです。

### どのように使用しますか？

インターフェースには、企業名、担当する派遣会社のオフィス、都道府県、電話番号、および締め日を含む顧客のマスターリストが表示されます。

リストから顧客をクリックすると、管理者は5つのナビゲーションタブを持つ堅牢なパネルに誘導されます：

1. **顧客情報:**
   - **登録データ:**
     - 企業名および担当オフィス；
     - 完全な住所（Google Maps URL統合付き）；
     - 電話およびFAX；
     - 締め日および支払日；
     - 自由なメモのためのスペース。
   - **勤怠通知:**
     - 提出物の迅速な監視；
     - 配属されている従業員の週次勤怠通知。

2. **部門:**
   - **部門リスト:**
     - 企業内のすべての稼働中の作業エリアのリスト。
   - **財務および契約詳細（選択した部門）:**
     - 要求される作業の説明；
     - 支払われる基本給（男性/女性）；
     - 残業、深夜割増、および休日出勤のパーセンテージ；
     - 使用されるカレンダー（国の暦、独自、または顧客の暦）に関するメモ；
     - 退職のルール（事前通知、契約上の罰金など）。
   - **勤務スケジュール:**
     - 始業時間；
     - 休憩および昼休み時間；
     - その部門で定義されている週の休日。
   - **責任者:**
     - 顧客企業内部の連絡先（マネージャー、ラインリーダーなど）；
     - その部門を直接担当する連絡先。

3. **配属された従業員:**
   - **アクティブな配属パネル:**
     - 部門のリストを含むカード；
     - 選択した部門の従業員の氏名リストを含むカード。

4. **勤怠アップロード:**
   - **タイムシート処理:**
     - 企業から送信された勤怠リストのアップロード；
     - 時間を直接計算するためのOCR（光学式文字認識）処理。

   <Callout title="実装フェーズ:">
     OCRの統合は現在開発段階にあります。一時的に、処理は半自動で運用されており、時間の計算を確定するには、管理者が勤怠データを手動で入力する必要があります。
   </Callout>

5. **請求（請求書）:**
   - **請求書モジュール:**
     - 企業向けの請求書生成；
     - 各従業員の締め切られた時間に基づく計算。

### 期待される結果

正確で堅牢なB2Bコントロール。派遣会社は、給与交渉、責任者とのコミュニケーション、および各顧客への請求を一元化し、給与計算と請求が各部門の契約上の合意を厳密に遵守することを保証します。
