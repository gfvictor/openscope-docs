# Templates

### What is it?

A centralized repository for storing and configuring file templates used by the system's export and document issuance engine.

### What is it for?

To standardize the official documents issued by the agency (such as resumes, contracts, and internal reports). The system captures predefined variables in these Excel files (`.xlsx`) and injects the actual data of the employees, autonomously generating high-visual-fidelity final documents.

### How is it used?

The interface operates as a direct file manager:

- **Upload Repository:**
  - The manager uploads structurally formatted spreadsheets (containing headers, corporate color palette, logos);
  - Configuration of tags and variables corresponding to the database fields (see the [Injection Tags](templates/tags) documentation).

- **Internal Management:**
  - Visualization panel of currently active templates for exporting resumes (Rirekisho) and professional profiles;
  - Administration of bureaucratic forms and documentation for strictly internal circulation.

### Expected Result

Corporate standardization and scalability. The agency mitigates the manual filling of forms in Excel, ensuring that all external documentation strictly follows the brand's visual standard, with consolidated data free from human error.
