# 勤怠データの入力

### どのような機能ですか？

管理者が従業員の労働時間を入力（または確認）し、給与計算の自動化を可能にする運用統合モジュールです。

### 何のためのものですか？

物理的なタイムカードや顧客のスプレッドシートに記録されたデータをシステムのデータベースに転記し、正確な給与計算と請求書作成のために正しい契約変数（残業、深夜割増、休日）を適用することを保証するためです。

### どのように使用しますか？

対象の顧客企業を選択した後、インターフェースには2つのセクションに分かれた入力パネルが表示されます：

1. **従業員パネル:**
   - 選択した企業でアクティブな配属を持つすべての従業員の氏名リスト；
   - 時間マトリックスをロードするための従業員の個別選択。

2. **勤怠マトリックス（現在のサイクル）:**
   - 現在の支払い期間のすべての日を含む動的テーブル。各日について、以下の指標フィールドが利用可能です：
     - サイクルの事前定義された日付；
     - 定時労働時間（Teiji）；
     - 1日の残業時間のボリューム；
     - 通常の深夜割増を伴う労働時間；
     - 深夜残業時間（深残）；
     - 休日または有給休暇に働いた時間。

<Callout title="OCRによる自動化（開発中）:">
  現在、このプロセスには手動でのデータ入力が必要です。OCRモジュールの本格的な統合後、これらのフィールドはタイムカードの光学式読み取りに基づいて自動的に入力されるようになります。管理者の役割は、確認と異常の監査のみに移行します。
</Callout>

<Callout title="データ処理:">
  入力中、給与計算はリアルタイムで実行されません。データが検証され、給与計算エンジンに注入されるように、マトリックスの入力が完了した後に管理者が「保存」アクションを実行することが必須です。
</Callout>

### 期待される結果

会計上の正確性。派遣会社は給与計算の入力エラーを軽減し、労働法の複雑な変数を統合し、請求の準備が整った監査可能なデータベースに勤怠データを集中化させます。
