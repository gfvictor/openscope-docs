# Waiting Hire

### What is it?

A monitoring list dedicated exclusively to collaborators who have already accepted an allocation but are waiting for their start date at the client company.

### What is it for?

To provide managers with a clear, centralized view of upcoming hires currently in the pipeline ("on hold"), ensuring that new employees are not forgotten between the client's approval and their actual first day of work.

### How is it used?

The interface displays detailed individual panels for each upcoming allocation.

1. **Allocation Information Panel:**
   - Allocation acceptance date;
   - Employee's full name;
   - Client company and destination sector;
   - Job type (Full-time, Part-time, etc.);
   - Scheduled start date;
   - Salary paid to the employee and salary type (hourly, daily, or fixed monthly);
   - Name of the manager responsible for the initial allocation;
   - Name of the manager who confirmed the allocation and the respective date;
   - Current allocation status.

2. **Management and Editing:**
   - **Status Change:** The manager can instantly change the allocation status using the quick control button on the panel itself;
   - **Advanced Editing:** Clicking the employee's name opens the full professional profile (the same one detailed in the Candidates section), allowing the manager to adjust any registration or contractual data before the start date.

### Expected result

An organized, frictionless transition between recruitment and the actual start of work. Managers can approve, pause, or reject imminent contracts with just a few clicks, maintaining full control over the agency's onboarding flow.
