# 退職待ち

サイクルの終了。社員は去りましたが、管理上の保留事項がまだ存在しています。

### どのような機能ですか？

辞任したか解雇されたが、システム上で退職プロセスがまだ完全に完了していない社員のキューです。

### 何のためのものですか？

最後の給与を支払う前に、保留事項（アパートの清掃、制服の返却、給与前払いの清算など）の徴収を忘れて派遣会社が経済的損失を被らないようにするためです。

### どのように使用しますか？

社員の退職を引き起こすと、そのプロフィールはこの画面で凍結されます。
人事または財務管理者はこのメニューにアクセスして、退職のチェックリストに目を通します：

- 社員がすでに寮（社宅）を退去しているか確認する。
- 派遣会社の備品の返却を確認する。
- 退職金の計算を実行する。
  これらの保留事項すべてに「チェック」を入れた後でのみ、管理者は退職を完了としてマークし、プロフィールを完全にアーカイブし、（寮と配属先の）枠を解放します。

### 期待される結果

未解決事項の解消。派遣会社は、家賃や前払いを借りたまま姿を消す社員から損失を被ることはなく、合法的および事務的な退職フロー全体が遵守され、文書化されることを保証します。
