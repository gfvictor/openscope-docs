# Leaves and Vacations

### What is it?

An administrative dashboard to view, monitor, and respond to all absence requests made by employees.

### What is it for?

To centralize requests for days off, vacations, or delays, allowing the agency and the end client to adequately plan for the employee's absence, keeping the attendance history up-to-date and preventing gaps in the work schedule.

### How is it used?

The interface displays a control list with all requests, containing the following information:

1. **Request Details:**
   - Employee's name;
   - Client company where they are allocated;
   - Requested usage date(s);
   - Type of absence (Vacation, Leave, or Delay);
   - Volume of hours (Morning, Afternoon, or Full day);
   - Current request status;
   - Name of the manager who evaluated the request and the respective confirmation date.

2. **Request Management:**
   - Through a direct button on the list panel, the manager can instantly change the request status to **Approved**, **Rejected**, or **Canceled**.

### Expected Result

Organization and predictability. The agency resolves absence requests in an agile and documented manner, ensuring transparency for the employee and operational security for the client company.
